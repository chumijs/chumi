# 贡献指南

1. 核心代码在[`chumi`](./chumi/)
	```sh
	# 安装依赖
	$ yarn
	```

2. 基于单元测试用例进行功能开发和调试，所有测试代码和功能，都在[`test`](./test/)

	```sh
	$ yarn test
	```

3. 请遵循如下规范

> 1. 你可以通过编写测试用例（可以参考已有的`测试用例`来编写）,来测试你的新功能，或者修复的bug
> 
> 2. 每个贡献，都必须对其进行完善的单元测试，同时尽可能进行**功能可用性**测试，而*非单纯的代码可用性测试*
>
> 3. 保证所有的代码都是使用**TypeScript**编写，同时尽量`避免使用any`
